# 🖼️🔐 PicSecret 图密 — 图片隐写通信工具

[![Deploy to GitHub Pages](https://github.com/actions/workflows/pages/pages-build-deployment/badge.svg)](../../actions)

一个**纯前端**的 PNG 图片隐写通信工具，可以把文本秘密隐藏在 PNG 图片中，通过图片传递加密信息。

---

## ✨ 功能特性

| 功能 | 说明 |
|------|------|
| 🔒 **信息隐藏** | 将任意文本嵌入 PNG 图片的 tEXt 数据块中，图片外观完全不变 |
| 🔐 **AES-256 加密** | 可选口令加密，使用 PBKDF2 + AES-256-GCM 保护内容 |
| 🔓 **信息提取** | 上传图片即可自动检测并提取隐藏信息 |
| 📱 **响应式设计** | 手机/平板/电脑全适配 |
| 🌐 **中英双语** | 一键切换中文/English |
| ⚡ **纯前端** | 无需后端服务器，数据完全在浏览器本地处理 |
| 🚀 **GitHub Pages** | 可直接部署到 GitHub Pages，全球访问 |

---

## 🚀 快速部署（GitHub Pages）

### 方式一：自动部署（推荐）

1. Fork 或上传本仓库到你的 GitHub
2. 进入仓库 **Settings → Pages**
3. **Source** 选择 `GitHub Actions`
4. 推送代码到 `main` 分支，自动触发部署
5. 部署完成后访问 `https://你的用户名.github.io/仓库名/`

### 方式二：手动部署

1. 进入仓库 **Settings → Pages**
2. **Source** 选择 `Deploy from a branch`，分支选 `main`，目录选 `/ (root)`
3. 保存后等待部署完成

### 本地运行

直接用浏览器打开 `index.html` 即可，无需任何服务器。

## 📖 使用方法

### 编码（隐藏信息）

1. **上传载体图片** — 选择一张 PNG 图片作为「信封」
2. **输入秘密信息** — 在文本框中输入想隐藏的内容
3. **设置口令（可选）** — 设置密码后信息将被 AES-256 加密
4. **点击编码** — 生成带有隐藏信息的新图片
5. **下载发送** — 下载图片，发给接收者

### 解码（查看信息）

1. **上传加密图片** — 拖入含有隐藏信息的 PNG 图片
2. **自动检测** — 工具会自动识别是否含有隐藏信息
3. **输入口令** — 如果信息已加密，输入正确口令
4. **查看内容** — 隐藏信息将显示出来，可一键复制

## ⚠️ 重要提示

| ✅ 可以 | ❌ 不可以 |
|---------|-----------|
| 邮件附件发送 | 微信/QQ 发「图片」（会压缩） |
| 网盘/云存储分享 | 社交媒体发布（会压缩） |
| 即时通讯「原图/文件」模式 | 截图后再发送 |
| 蓝牙/AirDrop 传输 | 经过图片压缩工具处理 |

> 💡 **原理**：信息存储在 PNG 文件的元数据块中。任何对图片的重新压缩都会清空这些元数据。

---

## 🔧 技术原理

- **PNG tEXt 数据块** — PNG 规范允许在文件中嵌入任意文本元数据，任何图片查看器都能正常打开
- **AES-256-GCM** — 使用 Web Crypto API 进行浏览器端加密
- **PBKDF2** — 密钥派生函数，10 万次迭代，防止暴力破解
- **纯前端** — 所有处理都在浏览器中完成，图片不会上传到任何服务器
